# 贪吃蛇游戏完整版 — 项目构建方案

## 项目概述

使用 **Python + Pygame** 构建一款功能完整的经典贪吃蛇游戏。支持多种食物类型、障碍物、难度递增、连击倍率、排行榜、皮肤切换和音效系统。

## 技术选型

| 项目 | 选择 | 理由 |
|------|------|------|
| 语言 | Python 3.12+ | 简洁易读，适合游戏原型开发 |
| 游戏框架 | Pygame 2.6+ | 成熟的 2D 游戏库，社区活跃 |
| 依赖管理 | uv | 极速依赖安装，替代 pip/poetry |
| 测试框架 | pytest | 功能强大，生态丰富 |

## 核心架构

```
┌─────────────────────────────────────────────┐
│                  游戏引擎                      │
│            (主循环 + 场景状态机)                │
├──────────┬──────────┬──────────┬─────────────┤
│  输入层   │  实体层   │  系统层   │   渲染层     │
│ 按键映射  │ 蛇/食物   │ 碰撞检测  │ 游戏区域绘制  │
│ 动作枚举  │ 障碍物    │ 计分连击  │ UI面板绘制    │
├──────────┴──────────┴──────────┴─────────────┤
│              基础设施层                         │
│    配置管理 │ 皮肤管理 │ 音效管理 │ 排行榜持久化   │
└──────────────────────────────────────────────┘
```

**设计思路：**

- **场景状态机** 驱动 UI 流转（菜单 → 游戏 → 暂停 → 结束），每个场景独立封装，互不耦合
- **引擎** 统一管理主循环、帧率控制和场景切换，是唯一的"指挥官"
- **实体层** 定义游戏对象（蛇、食物、障碍物）的数据和行为
- **系统层** 实现游戏规则（碰撞、计分、难度），与实体层解耦
- **渲染层** 纯粹负责画面输出，不关心游戏逻辑

## 游戏功能

### 核心玩法
- 蛇在 20×20 网格中移动，吃食物变长、得分
- 碰墙、碰自己、碰障碍物均扣生命值
- 初始 3 条生命，生命归零则游戏结束
- 死亡后蛇重生到中央位置，获得 2 秒无敌时间

### 食物类型（4 种）
| 类型 | 分值 | 特殊效果 |
|------|------|----------|
| 🔴 普通 | 10 分 | 无 |
| 🟡 奖励 | 30 分 | 限时 8 秒后消失 |
| 🔵 减速 | 10 分 | 临时降低蛇速 5 秒 |
| 🩷 生长 | 20 分 | 蛇额外增长 2 节 |

### 难度系统
- 每获得 50 分自动升级
- 等级提升 → 蛇移动速度加快（8步/秒 → 最高20步/秒）
- 2 级起每级增加 3 个障碍物

### 计分系统
- 连续吃食物触发**连击倍率**（第2次 ×2，第3次 ×3…）
- 死亡或碰撞后连击清零
- 排行榜记录前 10 名，JSON 持久化存储

### 皮肤系统（3 套）
- **经典** — 绿色蛇身 + 灰色背景
- **复古** — 荧光绿 + 纯黑背景
- **霓虹** — 青蓝蛇身 + 深蓝背景

### 音效系统
- 吃食物、死亡、菜单选择等事件触发音效
- 支持音量调节和静音开关

## 目录结构

```
SnakeGame/
├── main.py              # 入口（极简，仅启动引擎）
├── pyproject.toml       # 项目配置
├── config/              # 配置文件（皮肤定义、用户设置、排行榜）
├── src/
│   ├── core/            # 核心：引擎、状态枚举、配置、输入
│   ├── entities/        # 实体：蛇、食物、障碍物
│   ├── systems/         # 系统：碰撞、计分、难度
│   ├── rendering/       # 渲染：游戏区域、UI面板
│   ├── ui/              # 场景：菜单、游戏、暂停、结束、设置等
│   ├── skins/           # 皮肤管理
│   └── audio/           # 音效管理
├── assets/              # 资源：音效文件、字体
├── scripts/             # 脚本：run.sh、test.sh
├── tests/               # 测试文件
└── docs/                # 文档
```

**设计原则：**
- 每个文件不超过 300 行
- 每层目录不超过 8 个文件
- 模块间单向依赖，无循环引用

## 实施路径

| 阶段 | 任务 | 产出 |
|------|------|------|
| **第 1 阶段** | 脚手架 + 配置 + 输入 | 可运行的项目骨架，测试通过 |
| **第 2 阶段** | 蛇 + 食物 + 障碍物 + 碰撞 | 全部核心实体和碰撞检测 |
| **第 3 阶段** | 计分 + 难度系统 | 游戏规则完整 |
| **第 4 阶段** | 引擎 + 渲染 + 场景 | 可玩的完整游戏 |
| **第 5 阶段** | 音效 + 集成测试 | 最终交付 |

每个阶段遵循 **TDD 流程**：先写测试 → 确认失败 → 写实现 → 确认通过 → 提交。

## 运行方式

```bash
# 启动游戏
bash scripts/run.sh

# 运行测试
bash scripts/test.sh
```
